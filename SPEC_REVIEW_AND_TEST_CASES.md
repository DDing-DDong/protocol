# Trace Protocol 명세 리뷰 및 테스트케이스

작성일: 2026-07-09  
기준 코드: `js/data.js`, `js/stage.js`, `js/replay.js`, `js/game.js`

## 1. 명세 리뷰 요약

현재 구현은 **공격 턴에서 해커 경로를 기록하고, 방어 턴에서 해당 경로를 리플레이하며, 제한된 함정 배치로 방어 목표를 달성하는 2턴 순환 게임**으로 정리된다. 핵심 검증 대상은 리플레이 재현성, 함정 판정, 방어 목표 달성 판정, 보상 효과의 적용 범위, 스테이지 전환이다.

## 2. 현재 구현 기준 핵심 명세

### 2.1 턴과 스테이지

- 홀수 스테이지는 공격 턴, 짝수 스테이지는 방어 턴이다.
- 턴 값은 `attack`, `defense_build`, `defense_replay`, `ending` 네 가지다.
- 스테이지 11 클리어 후 엔딩 선택지가 뜨며, “흑막을 추적한다” 선택 시 스테이지 12 무한 모드로 진입한다.
- 방어 턴 시작 시 이전 공격 기록이 2개 미만이면 이전 스테이지로 되돌아가 다시 세팅된다.

### 2.2 공격 턴

- 공격 턴 목표는 데이터 코어 도달이다.
- UI 도움말 기준 조작은 방향키 좌/우 이동, 위 점프, `Shift` 대시, `Space` 보호막/해킹이다.
- 공격 기록은 `sampleStep` 간격으로 저장되며 현재 구현값은 0.06초다.
- 기록 샘플은 시간, 위치, 속도, 방향, 지상 여부, 슬라이딩 여부, 보호막 상태, 누적 에너지 사용량을 포함한다.

### 2.3 방어 턴

- 방어 준비 턴에서 함정을 배치한 뒤 리플레이를 시작한다.
- 리플레이는 이전 공격 기록을 순차 재생하며, 재생 속도는 1.5배다.
- 방어 성공은 `getDefenseObjectiveItems(game)`의 모든 항목이 완료됐을 때 성립한다.
- 방어 실패 시 같은 방어 스테이지를 재시도하며, 실패한 함정 배치는 유지된다.

### 2.4 함정

구현상 함정 종류와 기본 비용:

| 함정 | 비용 | 구현 효과 |
| --- | ---: | --- |
| 레이저 | 2 | 탐지 +1, 강화 시 탐지 +2 |
| 감전패널 | 2 | 리플레이 지연, 강화/보상에 따라 지연 증가 |
| 카메라 | 1 | 탐지 +1, 경보 충전, 다음 함정 강화, 보상 시 짧은 지연 가능 |
| 방화벽 | 1 | 강화 또는 닫힘 상태일 때 지연 |
| EMP패널 | 1 | 에너지 흡수 20, 강화 시 30 |

슬라이딩 중인 리플레이 해커는 감전패널과 EMP패널을 통과할 수 있다.

### 2.5 방어 목표

현재 스토리 방어 목표는 코드 기준 다음과 같다.

| 스테이지 | 목표 |
| ---: | --- |
| 2 | 지연 2초, 에너지 흡수 20, 함정 2개 이하 |
| 4 | 탐지 2회, 지연 6초, 함정 3개 이하, 레이저 사용 |
| 6 | 탐지 2회, 지연 1.8초, 함정 3개 이하, 카메라/감전패널 사용 |
| 8 | 탐지 2회, 지연 1.8초, 에너지 흡수 20, 함정 4개 이하, 카메라/감전패널/EMP 사용 |
| 10 | 탐지 2회, 지연 4초, 함정 4개 이하, 카메라/방화벽 사용 |

`energyDrained` 목표가 있으면 EMP 사용 요구가 자동 추가된다. 지연 목표가 4초 이상이고 방화벽이 요구되면 카메라 사용 요구도 자동 추가된다.

## 4. 테스트 범위와 우선순위

### P0: 게임 성립을 막는 핵심

- 공격 기록 생성 및 방어 턴 리플레이 시작
- 리플레이 경로 재현
- 함정 판정과 목표 완료
- 실패/성공 시 스테이지 전환
- 방어 실패 후 배치 유지

### P1: 밸런스와 진행감

- 스테이지별 목표 표시와 실제 목표 일치
- 함정 비용, 예산, 개수 제한
- 보상 적용 및 만료
- 무한 모드 난이도 증가

### P2: 사용성/표시

- 튜토리얼, 도움말, 로그 문구
- 모바일/가로 모드 UI
- BGM/SFX 상태 전환
- 로비, 설정, 재시작, 나가기

## 11. 회귀 테스트 체크리스트

릴리즈 전 최소 수동 회귀는 다음 순서로 수행한다.

1. 로비에서 시작해 스테이지 1 튜토리얼을 넘긴다.
2. 스테이지 1을 클리어하고 보상 선택 후 스테이지 2로 이동한다.
3. 스테이지 2에서 함정 2개 이하로 방어 목표를 성공시킨다.
4. 스테이지 3 공격 턴에서 이전 방어 함정이 위험 요소로 반영되는지 확인한다.
5. 방어 실패를 1회 유도해 배치 유지와 재도전 흐름을 확인한다.
6. 스테이지 11 클리어 후 두 엔딩 분기 중 무한 모드 진입을 확인한다.
7. 스테이지 12 이후 최고 기록 저장을 확인한다.
